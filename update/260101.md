# WordEcho 26.1.2 更新日志

## 新增

- **公告推送系统**：应用启动时自动拉取公告，支持多条公告按队列逐个弹出，包含锁定阅读、不再提醒、过期筛选等完整功能。公告支持内联 `Markdown` 内容或远程 MD 文件加载，按发布时间排序，过期自动过滤
- **更新检测与下载引导**：启动时自动检查新版本，支持手动触发检查（关于页面刷新按钮），强制更新时不可跳过，非强制更新可选择跳过当前版本
- **跳过此版本**：非强制更新可选择跳过当前版本，后续不再提示该版本，新版本发布后自动恢复提示
- **Markdown 渲染引擎**：自研轻量 MD 渲染器，支持标题、有序/无序列表、代码块、引用、行内代码、粗体、斜体、删除线、链接、分隔线等完整语法
- **壁纸收藏功能**：浏览每日 Bing 壁纸时可收藏喜欢的壁纸，收藏列表支持批量管理
- **历史公告页**：查看所有历史公告记录，支持按时间排序和筛选

## 详细说明

### 公告系统架构

公告数据存储在 `WordEcho-Release` 仓库，使用 `announcements.csv` 作为元数据索引，每个公告对应一个 `announcement/{id}.md` 文件。客户端启动后拉取 CSV，解析后按以下条件过滤：

1. `enabled` 为 `false` 跳过
2. `publishDate` 未到跳过
3. `expireDate` 已过跳过
4. `minVersionCode` / `maxVersionCode` 版本不匹配跳过
5. `targetGroups` 不包含当前用户组跳过
6. 已标记"不再提醒"的跳过

> 多条公告按数组顺序逐个弹出，前一条关闭后才弹下一条。锁定阅读期间关闭按钮不可点击。

### 更新检测流程

更新配置存储在 `update.json` 中，包含版本号、下载地址、是否强制更新等字段。客户端启动时拉取并与本地版本比对：

- `versionCode` 大于本地版本 -> 有更新
- 被用户跳过的版本 -> 不提示
- 强制更新 -> 不可关闭对话框，不可跳过
- 非强制更新 -> 可跳过此版本或稍后提醒

### 兼容性

- 最低支持 Android 8.0 (API 26)
- 推荐 Android 12+ 以获得最佳 Toast 通知体验
- 手环连接需支持 `InterHandshake` 协议
